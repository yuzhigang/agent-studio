Agent Studio Runtime Architecture

Event-Driven Multi-World Simulation Platform

Event-Driven Design

  • All communication via EventBus
  • No direct instance-to-instance calls
  • Scope isolation: world vs scene
  • Replayable event log for recovery

Scene & Instance Model

  • Shared mode: references existing instances
  • Isolated mode: copy-on-write deep copies
  • Instance behavior from Model definitions
  • AlarmManager reuses trigger system

State & Security

  • Checkpoint every 30s + event replay
  • SQLite per-world persistence
  • SandboxExecutor with safe builtins
  • Forbidden: open, eval, exec, __import__

设计要点

  1. Supervisor 是单例进程，Worker 是多实例进程，Worker 启动时通过 JSON-RPC 向 Supervisor 注册并汇报数据。
  2. Worker 是独立运行的进程，通过多 Channel 接受外部 Message，并由 MessageHub 路由到多个 World 实例。
  3. Worker 管理多个 World 实例，负责 World 的创建、持久化、启停和销毁。
  4. World 是多智能体同时运行的环境，内部通过 EventBus 进行消息通信，同时可直接调用实例 Service 改变状态。
  5. Scene 是 World 下多个智能体的聚合：Shared 模式保持对 World 实例的引用；Isolated 模式获得实例状态拷贝，隔离模式下不影响真实 World 状态。
  6. 脱离 Supervisor 亦可通过 CLI 对 Worker 进行部署和管理。
  7. 驱动 World 实例变迁的动力：物理世界数据点变化、外部 Channel 发送的 Message、实例自身定时任务。
